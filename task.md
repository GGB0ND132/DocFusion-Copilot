# DocFusion Copilot 任务账本

更新时间：2026-04-15

本文件是项目任务状态主账本。README 只保留能力入口，状态以本文件为准。

## 一、当前里程碑

- 已具备比赛主链路：文档解析 -> 事实抽取 -> 模板回填 -> 结果下载。
- 已具备 Agent 主链路：问答/摘要/提取/回填/导出。
- 已具备测试集按用户要求生成脚本。
- 已支持多任务并发回填与独立进度追踪。

## 二、近期已完成

1. 多任务并发回填：前端 `fillTaskIds[]` / `fillTasks Record` 替代旧单任务状态，可同时执行多个模板回填。
2. 「任务」Tab：替代原「追溯」Tab，展示所有回填任务卡片（状态、进度条、耗时、下载）。
3. 纯秒制耗时显示：回填任务时间统一为纯秒（如 `67.6 s`），去掉分秒混合格式。
4. 会话切换保留任务：切换对话时不再清空 `fillTasks`，任务进度持续可见。
5. 文档选择增强：选择对话框展示所有已解析源文档（按模板名模糊排序），支持搜索过滤与全选/取消全选。
6. 移除 Docker：删除 Compose 与 Dockerfile，简化为本地启动。
7. pgvector 修复：修正嵌入维度与索引创建逻辑，确保向量检索可用。
8. Agent 跳过模板回填：template_fill 步骤由专用服务直接执行，不经 Agent 调度。
9. 移除文档选择中的 embedding/匹配度列，简化界面。
10. 测试集脚本支持三场景（城市经济 / 山东空气 / COVID）。

## 四、验收口径

- 准确率目标：模板回填准确率 >= 80%。
- 时延目标：单模板回填 <= 90 秒。
- 追溯目标：填充值可追溯到来源文档与证据片段。

## 五、执行命令速查

后端启动：

```bash
cd backend
python app/main.py
```

前端启动：

```bash
cd frontend
npm run dev
```

后端测试：

```bash
cd backend
python -m pytest tests -v
```

测试集结果生成：

```bash
cd backend
python generate_testset_by_requirements.py
```
